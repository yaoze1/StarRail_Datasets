# 财务格式化与输出标准 — 完整 Agent 指南

> 本文档是 agent 对 xlsx 文件应用专业财务格式时的完整参考手册。所有操作针对 `xl/styles.xml` 的直接 XML 手术，不使用 openpyxl。每个操作步骤都提供即用 XML 片段。

---

## 1. 何时使用此路径

本文档（FORMAT 路径）适用于以下两种场景：

**场景 A — 已有文件的专门格式化**
用户提供已有 xlsx 文件，要求应用或统一财务建模格式标准。起点是解包文件、审计现有 `styles.xml`，然后追加缺失样式并批量更新单元格 `s` 属性。不修改单元格值或公式。

**场景 B — CREATE/EDIT 后应用格式标准**
完成数据录入或公式编写后，格式化作为最后步骤应用。此时 `styles.xml` 可能来自 minimal_xlsx 模板（预定义 13 个样式槽）或来自用户文件。无论哪种，遵循"仅追加，绝不修改已有 xf 条目"原则。

**不适用**：仅读取或分析文件内容（用 READ 路径）；修改公式或数据（用 EDIT 路径）。

## 2. 财务格式语义系统

### 2.1 字体颜色 = 单元格角色（颜色 = 角色）

财务建模的首要约定：**字体颜色编码单元格角色，而非装饰**。审阅者可扫一眼颜色确定哪些单元格是可调参数、哪些是模型计算结果。这是全行业约定（投行、四大、企业财务团队遵循）。

| 角色 | 字体颜色 | AARRGGBB | 用例 |
|------|-----------|----------|----------|
| 硬编码输入/假设 | 蓝 | `000000FF` | 增长率、折现率、税率及其他用户可调参数 |
| 公式/计算结果 | 黑 | `00000000` | 所有含 `<f>` 元素的单元格 |
| 同工作簿跨表引用 | 绿 | `00008000` | 公式以 `SheetName!` 开头的单元格 |
| 外部文件链接 | 红 | `00FF0000` | 公式含 `[FileName.xlsx]` 的单元格（标记为脆弱链接） |
| 标签/文本 | 黑（默认） | 主题色 | 行标签、类别标题 |
| 需审阅的关键假设 | 蓝字体 + 黄填充 | 字体 `000000FF` / 填充 `00FFFF00` | 临时值、待确认参数 |

**决策树**：
```
单元格是否含 <f> 元素？
  +-- 是 -> 公式是否以 [FileName] 开头？
  |           +-- 是 -> 红（外部链接）
  |           +-- 否  -> 公式是否含 SheetName!？
  |                       +-- 是 -> 绿（跨表引用）
  |                       +-- 否  -> 黑（同表公式）
  +-- 否  -> 值是否为用户可调参数？
              +-- 是 -> 蓝（输入/假设）
              +-- 否  -> 黑默认（标签）
```

**严禁**：蓝字体 + `<f>` 元素共存（颜色角色矛盾 — 必须纠正）。

### 2.2 数字格式矩阵

| 数据类型 | formatCode | numFmtId | 显示示例 | 适用场景 |
|-----------|-----------|----------|-----------------|---------------------|
| 标准货币（整元） | `$#,##0;($#,##0);"-"` | 164 | $1,234 / ($1,234) / - | 损益表、资产负债表金额行 |
| 标准货币（带分） | `$#,##0.00;($#,##0.00);"-"` | 169 | $1,234.56 / ($1,234.56) / - | 单价、明细成本 |
| 千（K） | `#,##0,"K"` | 171 | 1,234K | 管理报告简化显示 |
| 百万（M） | `#,##0,,"M"` | 172 | 1M | 宏观汇总行 |
| 百分比（1 位小数） | `0.0%` | 165 | 12.5% | 增长率、毛利率 |
| 百分比（2 位小数） | `0.00%` | 170 | 12.50% | IRR、精确利率 |
| 倍数/估值乘数 | `0.0x` | 166 | 8.5x | EV/EBITDA、P/E |
| 整数（千位分隔） | `#,##0` | 167 | 12,345 | 员工数、单位数量 |
| 年份 | `0` | 1（内置，无需声明） | 2024 | 列标题年份，防止 2,024 |
| 日期 | `m/d/yyyy` | 14（内置，无需声明） | 3/21/2026 | 时间线 |
| 常规文本 | General | 0（内置，无需声明） | — | 标签行、无格式要求的单元格 |

numFmtId 169–172 是自定义格式，需在 minimal_xlsx 模板预定义的 4 个格式（164–167）之外追加。追加时按规则分配 ID（见第 3.4 节）。

**内置格式 ID 无需在 `<numFmts>` 中声明**（ID 0–163 是 Excel/LibreOffice 内置的；在 `<xf>` 中直接引用 numFmtId 即可）：

| numFmtId | formatCode | 说明 |
|----------|-----------|-------------|
| 0 | General | 常规格式 |
| 1 | `0` | 整数，无千位分隔（年份用此 ID） |
| 3 | `#,##0` | 千位分隔整数（无小数） |
| 9 | `0%` | 百分比整数 |
| 10 | `0.00%` | 两位小数百分比 |
| 14 | `m/d/yyyy` | 短日期 |

### 2.3 负数显示标准

财务报告对负数有两种主流约定 — 选一种并在整个工作簿中**保持一致**：

**括号样式（投行标准，外部交付物推荐）**

```
正：$1,234    负：($1,234)    零：-
formatCode: $#,##0;($#,##0);"-"
```

**红色减号样式（适合内部运营分析报告）**

```
正：$1,234    负：-$1,234（红）
formatCode: $#,##0;[Red]-$#,##0;"-"
```

规则：一旦确定样式，整个工作簿保持一致。不要在同一工作簿内混用两种负数显示样式。

### 2.4 零值显示标准

财务模型中，"0"和"无数据"语义不同，应视觉区分：

| 场景 | 推荐显示 | formatCode 第三段 |
|----------|-------------------|--------------------------|
| 稀疏矩阵（多数行有零值期） | 破折号 `-` | `"-"` |
| 数量计数（零本身有意义） | `0` | `0` 或省略 |
| 占位行（显式空） | 留空 | 不写入单元格 |

四段格式语法：`正数格式;负数格式;零值格式;文本格式`

零显示为破折号：`$#,##0;($#,##0);"-"`
零保留为 0：`#,##0;(#,##0);0`

## 3. styles.xml 手术操作

### 3.1 审计现有样式：理解 cellXfs 间接引用链

单元格的 `s` 属性指向 `cellXfs` 中的位置索引（0 起始），`cellXfs` 中每个 `<xf>` 条目通过 `fontId`、`fillId`、`borderId`、`numFmtId` 引用各自定义库。

引用链示意图：

```
单元格 <c s="6">
    | 按 0 起始索引查 cellXfs
cellXfs[6] -> numFmtId="164" fontId="2" fillId="0" borderId="0"
    |            |               |          |
numFmts         fonts[2]      fills[0]   borders[0]
id=164          color=00000000  (无填充)  (无边框)
$#,##0...       黑
```

审计步骤：

**第 1 步**：读 `<numFmts>`，记录所有声明的自定义格式及其 ID：
```xml
<numFmts count="4">
  <numFmt numFmtId="164" formatCode="$#,##0;($#,##0);&quot;-&quot;"/>
  <numFmt numFmtId="165" formatCode="0.0%"/>
  <numFmt numFmtId="166" formatCode="0.0x"/>
  <numFmt numFmtId="167" formatCode="#,##0"/>
</numFmts>
```
记录：当前最大自定义 numFmtId = 167，下一个可用 ID = 168。

**第 2 步**：读 `<fonts>`，按 0 起始索引列出每个 `<font>` 的颜色和样式：
```
fontId=0 -> 无显式颜色（主题默认黑）
fontId=1 -> color rgb="000000FF"（蓝，输入角色）
fontId=2 -> color rgb="00000000"（黑，公式角色）
fontId=3 -> color rgb="00008000"（绿，跨表引用角色）
fontId=4 -> <b/> + color rgb="00000000"（粗体黑，表头）
```

**第 3 步**：读 `<fills>`，确认 fills[0] 和 fills[1] 是规范强制的保留条目（绝不删除）：
```
fillId=0 -> patternType="none"（规范强制）
fillId=1 -> patternType="gray125"（规范强制）
fillId=2 -> 黄色高亮（若存在）
```

**第 4 步**：读 `<cellXfs>`，按 0 起始索引列出每个 `<xf>` 条目的组合：
```
索引 0 -> numFmtId=0,   fontId=0, fillId=0 -> 默认样式
索引 1 -> numFmtId=0,   fontId=1, fillId=0 -> 蓝字体常规（输入）
索引 5 -> numFmtId=164, fontId=1, fillId=0 -> 蓝字体货币（货币输入）
索引 6 -> numFmtId=164, fontId=2, fillId=0 -> 黑字体货币（货币公式）
...
```

**第 5 步**：验证所有 count 属性与实际元素数匹配（count 不匹配会导致 Excel 拒绝打开文件）。

### 3.2 安全追加新样式（黄金规则：仅追加，绝不修改已有 xf）

**绝不修改已有 `<xf>` 条目**。修改会影响所有已引用该索引的单元格，破坏现有格式。只在末尾追加新条目。

追加新样式的完整原子操作序列（5 步都必须执行）：

**第 1 步**：确定是否需要新 `<numFmt>`

内置格式（ID 0–163）跳过此步。自定义格式追加到 `<numFmts>` 末尾：
```xml
<numFmts count="5">  <!-- count +1 -->
  <!-- 保持现有条目不变 -->
  <numFmt numFmtId="164" formatCode="$#,##0;($#,##0);&quot;-&quot;"/>
  <numFmt numFmtId="165" formatCode="0.0%"/>
  <numFmt numFmtId="166" formatCode="0.0x"/>
  <numFmt numFmtId="167" formatCode="#,##0"/>
  <!-- 新追加 -->
  <numFmt numFmtId="168" formatCode="$#,##0.00;($#,##0.00);&quot;-&quot;"/>
</numFmts>
```

**第 2 步**：确定是否需要新 `<font>`

检查现有字体是否已含匹配的颜色+样式组合。若无，追加到 `<fonts>` 末尾：
```xml
<fonts count="6">  <!-- count +1 -->
  <!-- 保持现有条目不变 -->
  ...
  <!-- 新追加：红字体（外部链接角色），新 fontId = 5 -->
  <font>
    <sz val="11"/>
    <name val="Calibri"/>
    <color rgb="00FF0000"/>
  </font>
</fonts>
```
新 fontId = 追加前的 count 值（原 count=5 时，新 fontId=5）。

**第 3 步**：确定是否需要新 `<fill>`

若需新背景色，追加到 `<fills>` 末尾（注意：fills[0] 和 fills[1] 绝不可修改）：
```xml
<fills count="4">  <!-- count +1 -->
  <fill><patternFill patternType="none"/></fill>       <!-- 0: 规范强制 -->
  <fill><patternFill patternType="gray125"/></fill>    <!-- 1: 规范强制 -->
  <fill>                                               <!-- 2: 黄色高亮 -->
    <patternFill patternType="solid">
      <fgColor rgb="00FFFF00"/>
      <bgColor indexed="64"/>
    </patternFill>
  </fill>
  <!-- 新追加：浅灰填充（预测期区分），新 fillId = 3 -->
  <fill>
    <patternFill patternType="solid">
      <fgColor rgb="00D3D3D3"/>
      <bgColor indexed="64"/>
    </patternFill>
  </fill>
</fills>
```

**第 4 步**：在 `<cellXfs>` 末尾追加新 `<xf>` 组合
```xml
<cellXfs count="14">  <!-- count +1 -->
  <!-- 保持现有条目 0-12 不变 -->
  ...
  <!-- 新追加索引=13：带分货币公式（黑字体 + numFmtId=168） -->
  <xf numFmtId="168" fontId="2" fillId="0" borderId="0" xfId="0"
      applyFont="1" applyNumberFormat="1"/>
</cellXfs>
```
新样式索引 = 追加前的 count 值（原 count=13 时，新索引=13）。

**第 5 步**：记录新样式索引；随后将工作表 XML 中对应单元格的 `s` 属性设为此值。

### 3.3 AARRGGBB 颜色格式说明

OOXML 的 `rgb` 属性用 **8 位十六进制 AARRGGBB** 格式（非 HTML 的 6 位 RRGGBB）：

```
AA  RR  GG  BB
|   |   |   |
Alpha 红 绿 蓝
```

- Alpha 通道：`00` = 完全不透明（正常用值）；`FF` = 完全透明（不可见，绝不用）
- 财务颜色标准始终用 `00` 作 Alpha 前缀

| 颜色 | AARRGGBB | 对应角色 |
|-------|----------|-------------------|
| 蓝（输入） | `000000FF` | 硬编码假设 |
| 黑（公式） | `00000000` | 计算结果 |
| 绿（跨表引用） | `00008000` | 同工作簿跨表 |
| 红（外部链接） | `00FF0000` | 引用其他文件 |
| 黄（需审阅填充） | `00FFFF00` | 关键假设高亮 |
| 浅灰（预测期填充） | `00D3D3D3` | 区分历史与预测期 |
| 白 | `00FFFFFF` | 纯白填充 |

**常见错误**：误将 HTML 格式 `#0000FF` 写成 `FF0000FF`（Alpha=FF 使颜色完全透明不可见）。正确格式：`000000FF`。

### 3.4 numFmtId 分配规则

```
ID 0-163    -> Excel/LibreOffice 内置格式，无需在 <numFmts> 声明，在 <xf> 中直接引用
ID 164+     -> 自定义格式，必须在 <numFmts> 中显式声明为 <numFmt> 元素
```

分配新 ID 规则：
1. 读取当前 `<numFmts>` 中所有 `numFmtId` 属性值
2. 取最大值 + 1 作为下一个自定义格式 ID
3. 不复用已有 ID；不跳号

minimal_xlsx 模板预定义 ID：164、165、166、167。下一个可用 ID 为 168。

## 4. 预定义样式索引完整参考表（13 槽）

以下是 minimal_xlsx 模板 `styles.xml` 中预定义的 13 个样式槽（cellXfs 索引 0–12），可在工作表 XML 的单元格 `s` 属性中直接引用：

| 索引 | 语义角色 | 字体颜色 | 填充 | numFmtId | 格式显示 | 典型用途 |
|-------|--------------|------------|------|----------|---------------|-------------|
| **0** | 默认样式 | 主题黑 | 无 | 0 | General | 无需特殊格式的单元格 |
| **1** | 输入/假设（常规） | 蓝 `000000FF` | 无 | 0 | General | 文本型假设、标志 |
| **2** | 公式/计算结果（常规） | 黑 `00000000` | 无 | 0 | General | 文本拼接公式、非数字计算 |
| **3** | 跨表引用（常规） | 绿 `00008000` | 无 | 0 | General | 从跨表拉取的值（常规格式） |
| **4** | 表头（粗体） | 粗体黑 | 无 | 0 | General | 行/列标题 |
| **5** | 货币输入 | 蓝 `000000FF` | 无 | 164 | $1,234 / ($1,234) / - | 假设区的金额输入 |
| **6** | 货币公式 | 黑 `00000000` | 无 | 164 | $1,234 / ($1,234) / - | 模型区的金额计算（营收、EBITDA） |
| **7** | 百分比输入 | 蓝 `000000FF` | 无 | 165 | 12.5% | 假设区的比率输入（增长率、毛利率假设） |
| **8** | 百分比公式 | 黑 `00000000` | 无 | 165 | 12.5% | 模型区的比率计算（实际毛利率） |
| **9** | 整数（逗号）输入 | 蓝 `000000FF` | 无 | 167 | 12,345 | 假设区的数量输入（员工数） |
| **10** | 整数（逗号）公式 | 黑 `00000000` | 无 | 167 | 12,345 | 模型区的数量计算 |
| **11** | 年份输入 | 蓝 `000000FF` | 无 | 1 | 2024 | 列标题年份（无千位分隔） |
| **12** | 关键假设高亮 | 蓝 `000000FF` | 黄 `00FFFF00` | 0 | General | 待审阅或确认的关键参数 |

**选择指南**：
- 确定"输入"vs"公式" -> 选奇数（输入/蓝）或偶数（公式/黑）配对槽
- 确定数据类型 -> 选对应货币（5/6）/百分比（7/8）/整数（9/10）/年份（11）槽
- 需数字格式的跨表引用 -> 追加新绿+数字格式组合（见第 5.4 节）
- 待审阅参数 -> 索引 12

## 5. 假设分离原则：XML 层实现

### 5.1 结构设计

假设分离原则：**输入假设集中在专门区域（工作表或块）；模型计算区只含公式，无硬编码值**。

推荐结构：
```
工作簿工作表布局
  工作表 1 "Assumptions"  -> 所有蓝字体单元格（样式 1/5/7/9/11/12）
  工作表 2 "Model"        -> 所有黑或绿字体单元格（样式 2/3/4/6/8/10）
```

简单模型的同表分区法：
```
第 1-5 行：  [假设块 - 蓝字体]
第 6 行：    [空行分隔]
第 7+ 行：   [模型块 - 黑/绿字体公式引用假设区]
```

### 5.2 假设区 XML 示例

```xml
<!-- Assumptions 工作表（sheet1.xml）示例 -->

<!-- 第 1 行：块标题 -->
<row r="1">
  <c r="A1" s="4" t="inlineStr"><is><t>模型假设</t></is></c>
</row>

<!-- 第 2 行：增长率假设 - 蓝字体百分比输入，s="7" -->
<row r="2">
  <c r="A2" t="inlineStr"><is><t>营收增长率</t></is></c>
  <c r="B2" s="7"><v>0.08</v></c>
</row>

<!-- 第 3 行：毛利率假设 - 蓝字体百分比输入，s="7" -->
<row r="3">
  <c r="A3" t="inlineStr"><is><t>毛利率</t></is></c>
  <c r="B3" s="7"><v>0.65</v></c>
</row>

<!-- 第 4 行：基础营收 - 蓝字体货币输入，s="5" -->
<row r="4">
  <c r="A4" t="inlineStr"><is><t>基础营收（第 0 年）</t></is></c>
  <c r="B4" s="5"><v>1000000</v></c>
</row>

<!-- 第 5 行：关键假设（待审阅）- 蓝字体黄填充，s="12" -->
<row r="5">
  <c r="A5" t="inlineStr"><is><t>终值增长率</t></is></c>
  <c r="B5" s="12"><v>0.03</v></c>
</row>
```

### 5.3 模型区 XML 示例（引用假设区）

```xml
<!-- Model 工作表（sheet2.xml）示例 -->

<!-- 第 1 行：列标题（年份）- 粗体表头，s="4"；年份单元格，s="11" -->
<row r="1">
  <c r="A1" s="4" t="inlineStr"><is><t>指标</t></is></c>
  <c r="B1" s="11"><v>2024</v></c>
  <c r="C1" s="11"><v>2025</v></c>
  <c r="D1" s="11"><v>2026</v></c>
</row>

<!-- 第 2 行：营收行 -->
<row r="2">
  <c r="A2" t="inlineStr"><is><t>营收</t></is></c>
  <!-- B2：基年营收，从 Assumptions 跨表引用，绿，s="3"（常规格式） -->
  <!-- 若需货币格式，追加新样式 s="13"（见第 5.4 节） -->
  <c r="B2" s="3"><f>Assumptions!B4</f><v></v></c>
  <!-- C2、D2：次年营收 = 前年 * (1 + 增长率)，黑字体货币公式，s="6" -->
  <c r="C2" s="6"><f>B2*(1+Assumptions!B2)</f><v></v></c>
  <c r="D2" s="6"><f>C2*(1+Assumptions!B2)</f><v></v></c>
</row>

<!-- 第 3 行：毛利行 - 黑字体货币公式，s="6" -->
<row r="3">
  <c r="A3" t="inlineStr"><is><t>毛利</t></is></c>
  <c r="B3" s="6"><f>B2*Assumptions!B3</f><v></v></c>
  <c r="C3" s="6"><f>C2*Assumptions!B3</f><v></v></c>
  <c r="D3" s="6"><f>D2*Assumptions!B3</f><v></v></c>
</row>

<!-- 第 4 行：毛利率行 - 黑字体百分比公式，s="8" -->
<row r="4">
  <c r="A4" t="inlineStr"><is><t>毛利率 %</t></is></c>
  <c r="B4" s="8"><f>B3/B2</f><v></v></c>
  <c r="C4" s="8"><f>C3/C2</f><v></v></c>
  <c r="D4" s="8"><f>D3/D2</f><v></v></c>
</row>
```

### 5.4 追加"绿+数字格式"组合

预定义索引 3 是绿字体+常规格式。若跨表引用涉及货币金额，必须追加带数字格式的绿色样式：

```xml
<!-- 在 styles.xml 的 <cellXfs> 末尾追加（假设当前 count=13，新索引=13） -->
<!-- 索引 13：跨表引用 + 货币格式（绿字体 + $#,##0） -->
<xf numFmtId="164" fontId="3" fillId="0" borderId="0" xfId="0"
    applyFont="1" applyNumberFormat="1"/>
<!-- 更新 count 为 14 -->
```

追加后，跨表引用货币单元格用 `s="13"`。

## 6. 完整操作工作流

### 6.1 工作流概览

```
[已有 xlsx 或 CREATE/EDIT 后的文件]
        |
  第 1 步：解包（解压到临时目录）
        |
  第 2 步：审计 styles.xml（审查现有样式，构建索引映射表）
        |
  第 3 步：审计工作表 XML（识别需格式化的单元格及其语义角色）
        |
  第 4 步：追加缺失样式（numFmt -> font -> fill -> xf，更新计数）
        |
  第 5 步：批量更新工作表 XML 中各单元格的 s 属性
        |
  第 6 步：XML 有效性 + 样式引用完整性验证
        |
  第 7 步：打包（重新压缩为 xlsx）
```

### 6.2 第 1 步 — 解包

```bash
python3 SKILL_DIR/scripts/xlsx_unpack.py input.xlsx /tmp/xlsx_fmt/
```

若脚本不可用，手动解包：
```bash
mkdir -p /tmp/xlsx_fmt && cp input.xlsx /tmp/xlsx_fmt/input.xlsx
cd /tmp/xlsx_fmt && unzip input.xlsx -d unpacked/
```

### 6.3 第 2 步 — 审计 styles.xml

按第 3.1 节方法执行。minimal_xlsx 模板初始状态快速检查：
- `<cellXfs count="13">` 且 `<numFmts count="4">` -> 模板初始状态，13 个预定义槽可直接用
- 否则 -> 需完整审查现有索引映射

### 6.4 第 3 步 — 审计工作表 XML，构建格式化计划

读 `xl/worksheets/sheet*.xml`，评估每个单元格：
1. 是否含 `<f>` 元素（公式）？ -> 需黑/绿/红样式
2. 是否为硬编码数字参数？ -> 需蓝样式
3. 数据类型是货币/百分比/整数/年份？ -> 选对应数字格式槽
4. 是否为表头？ -> 粗体样式（索引 4）

构建格式化映射表：`{单元格坐标: 目标样式索引}`

### 6.5 第 4 步 — 追加样式

按第 3.2 节原子操作序列执行。每追加一个组件后立即更新对应 count 属性。

### 6.6 第 5 步 — 批量更新单元格 s 属性

```xml
<!-- 格式化前：无样式 -->
<c r="B5"><v>0.08</v></c>

<!-- 格式化后：增长率假设，蓝字体百分比，s="7" -->
<c r="B5" s="7"><v>0.08</v></c>
```

```xml
<!-- 格式化前：无样式公式 -->
<c r="C10"><f>B10*(1+Assumptions!B2)</f><v></v></c>

<!-- 格式化后：货币公式，黑字体，s="6" -->
<c r="C10" s="6"><f>B10*(1+Assumptions!B2)</f><v></v></c>
```

对于同类型连续行，可用行级默认样式减少重复：
```xml
<!-- 整行用样式=6，仅对例外单元格覆盖 -->
<row r="5" s="6" customFormat="1">
  <c r="A5" s="0" t="inlineStr"><is><t>营业利润</t></is></c>  <!-- 文本覆盖为默认 -->
  <c r="B5"><f>B3-B4</f><v></v></c>   <!-- 继承行级 s=6 -->
  <c r="C5"><f>C3-C4</f><v></v></c>
</row>
```

### 6.7 第 6 步 — 验证



## 7. 格式化完整性检查清单

交付前验证每项：

### 颜色角色一致性
- [ ] 所有含 `<f>` 元素的数字单元格：fontId 对应黑（公式）或绿（跨表引用）
- [ ] 所有用户可调参数的硬编码数字值：fontId 对应蓝（输入）
- [ ] 跨表引用（公式含 `SheetName!`）：fontId 对应绿
- [ ] 外部文件引用（公式含 `[FileName.xlsx]`）：fontId 对应红
- [ ] 无单元格同时含 `<f>` 元素并使用蓝字体（颜色角色矛盾）

### 数字格式正确性
- [ ] 年份列：numFmtId="1"（`0` 格式），显示为 2024 而非 2,024
- [ ] 货币行：numFmtId="164" 或变体，负数显示为 ($1,234) 而非 -$1,234
- [ ] 百分比行：值以小数存储（0.08 = 8%），格式 numFmtId="165"，显示为 8.0%
- [ ] 零值：在稀疏矩阵中显示为 `-` 而非 `0`（formatCode 第三段含 `"-"`）
- [ ] 倍数行（EV/EBITDA 等）：numFmtId="166"（`0.0x` 格式）
- [ ] 负数显示样式在整个工作簿一致（括号或红色减号）

### styles.xml 结构完整性
- [ ] `<numFmts count>` = 实际 `<numFmt>` 元素数
- [ ] `<fonts count>` = 实际 `<font>` 元素数
- [ ] `<fills count>` = 实际 `<fill>` 元素数（含规范强制的 fills[0] 和 fills[1]）
- [ ] `<cellXfs count>` = 实际 `<xf>` 元素数
- [ ] fills[0] 为 `patternType="none"`，fills[1] 为 `patternType="gray125"`（规范强制）
- [ ] 所有 `<xf>` 引用的 fontId / fillId / borderId 在各自集合有效范围内
- [ ] 所有单元格 `s` 属性值 < `cellXfs count`（无越界引用）

### 假设分离验证
- [ ] 假设区/表中无黑字体数字单元格（黑色数字 = 公式，不应在假设中）
- [ ] 模型区/表中无蓝字体非年份数字单元格（蓝色数字 = 硬编码，应在假设中）
- [ ] 模型区的输入参数通过公式引用假设区，而非直接复制值

### 公式与格式联动
- [ ] 所有含 `<f>` 元素的单元格都有显式 `s` 属性（不得用默认 style=0，其字体颜色非显式黑）
- [ ] SUM 汇总行：样式用黑字体 + 对应数字格式（如货币汇总用 s="6"）
- [ ] 百分比公式：值以小数存储，格式为 `0.0%`；不要在应用百分比格式前将值乘以 100

### 视觉层级
- [ ] 表头行（年份/指标名）：style=4（粗体黑）
- [ ] 汇总行（Total/EBITDA/Net Income）：粗体 + 对应数字格式（需要时追加样式）
- [ ] 单位说明行（如"$ 千"）：用 style=0 或 style=2（无需蓝）

## 8. 禁止操作（绝不可做）

- **不修改已有 `<xf>` 条目**：会批量改变所有引用该索引的单元格样式
- **不删除 fills[0] 和 fills[1]**：OOXML 规范要求；删除导致文件损坏
- **不修改单元格值或公式**：FORMAT 路径只改样式，不改内容
- **不用 openpyxl 格式化**：openpyxl 保存时重写整个 styles.xml，丢失不支持的功能
- **不应用全局覆盖样式**：不要用单一样式覆盖整个工作簿；按语义角色精确分配
- **不在 Alpha 通道写 FF**：`rgb="FF0000FF"` 使颜色完全透明；正确格式为 `rgb="000000FF"`

## 9. 常见错误与修复

### 错误 1：年份显示为 2,024

原因：年份单元格的 `s` 属性用了带千位分隔的格式（如 numFmtId="3" 或 numFmtId="167"）。

```xml
<!-- 错误 -->
<c r="B1" s="9"><v>2024</v></c>

<!-- 修复：改为 s="11"（numFmtId="1"，格式 0） -->
<c r="B1" s="11"><v>2024</v></c>
```

### 错误 2：百分比显示为 800%（值被乘以 100）

原因：8% 存为 `<v>8</v>` 而非 `<v>0.08</v>`。Excel 的 `%` 格式自动将值乘以 100 显示。

```xml
<!-- 错误 -->
<c r="B2" s="7"><v>8</v></c>

<!-- 修复：值必须以小数形式存储 -->
<c r="B2" s="7"><v>0.08</v></c>
```

### 错误 3：追加样式未更新 count 导致文件损坏

原因：追加了 `<font>` 或 `<xf>` 元素但未更新 count 属性；Excel 用旧 count 越界读取。

修复：每追加一个元素后立即更新对应 count：
```xml
<!-- 追加第 6 个字体后，count 必须从 5 改为 6 -->
<fonts count="6">
  ...
</fonts>
```

### 错误 4：蓝字体 + 公式（颜色角色矛盾）

原因：公式单元格误用输入样式（如货币输入 s="5"）。

```xml
<!-- 错误：公式单元格用蓝色输入样式 -->
<c r="C5" s="5"><f>B5*1.08</f><v></v></c>

<!-- 修复：公式单元格改为对应黑色公式样式（5->6, 7->8, 9->10） -->
<c r="C5" s="6"><f>B5*1.08</f><v></v></c>
```

### 错误 5：AARRGGBB 颜色缺 Alpha（仅 6 位）

```xml
<!-- 错误：6 位格式，行为取决于实现，通常导致颜色错误 -->
<color rgb="0000FF"/>

<!-- 修复：始终用 8 位 AARRGGBB，Alpha 固定 00 -->
<color rgb="000000FF"/>
```

### 错误 6：修改已有 xf（影响所有引用该索引的单元格）

原因：直接修改 cellXfs 中第 N 个 `<xf>` 的属性，导致所有 `s="N"` 的单元格被批量改变。

修复：保持现有条目不变，在末尾追加新条目，只将需新样式单元格的 `s` 属性改为新索引：
```xml
<!-- 错误：修改了索引=6 的已有 xf -->
<xf numFmtId="164" fontId="2" fillId="0" borderId="0" xfId="0"
    applyFont="1" applyNumberFormat="1" applyAlignment="1">
  <alignment horizontal="right"/>  <!-- 新增属性，影响所有已用 s="6" 的单元格 -->
</xf>

<!-- 修复：追加新索引（原 count=13 时，新索引=13），只改需右对齐单元格的 s 属性 -->
<!-- 索引=6 保持原样 -->
<xf numFmtId="164" fontId="2" fillId="0" borderId="0" xfId="0"
    applyFont="1" applyNumberFormat="1" applyAlignment="1">
  <alignment horizontal="right"/>
</xf>  <!-- 新索引=13 -->
```

## 10. 财务模型结构约定

### 10.1 表头行

- 粗体字体（对应本 skill 模板的样式索引 4）
- 年份列：用数字格式 `0`（numFmtId="1"，无千位分隔）防止 2024 显示为 2,024
- 表头下可加单位说明行：灰或斜体文本，如"$ 千"或"% 占营收"

### 10.2 行类型标准

| 行类型 | 样式建议 | 示例 |
|----------|---------------------|---------|
| 类别标题行 | 粗体，可选填充色 | "营收" |
| 明细项行 | 常规样式 | "产品 A"、"产品 B" |
| 小计行 | 粗体 + 顶边框 | "总营收" |
| 运营指标行 | 常规样式 | "毛利率 %" |
| 分隔行 | 空行 | (空) |

### 10.3 多年模型列布局

```
A 列：标签列          （宽 28，左对齐文本，表头 s="4" 或标签 s="0"）
B 列：FY2022 实际     （宽 12，年份表头 s="11"，数据单元格按语义角色样式）
C 列：FY2023 实际

E 列：FY2025E
F 列：FY2026E
```

参数从假设表传递到模型表的完整 XML 示例：

```xml
<!-- Assumptions 表，单元格 B5：8% 增长率，蓝色百分比输入 -->

<!-- Model 表，单元格 C10：引用假设区增长率，绿色百分比公式 -->
<!-- 需追加索引=13：绿 + 百分比格式（fontId=3, numFmtId=165） -->
<c r="C10" s="13"><f>Assumptions!B5</f><v></v></c>
```

## 11. 假设类别

在假设区（Assumptions 表或假设块），按以下标准顺序组织假设以便审阅和维护：

1. **营收假设**：增长率、定价、销量
2. **成本假设**：毛利率、固定/可变成本比率
3. **营运资金**：DSO（应收账款周转天数）、DPO（应付账款周转天数）、库存天数
4. **资本支出（CapEx）**：占营收百分比或绝对金额
5. **融资假设**：利率、债务偿还计划
6. **税务及其他**：有效税率、折旧摊销（D&A）

## 12. 审计追踪最佳实践

- 用 `s="12"`（蓝字体 + 黄填充高亮）标记需审阅或待改动的单元格，使其对审阅者立即可见
- 在敏感性分析行或单独的 Sensitivity 标签页，展示关键假设 +/-1% 变化对结果的影响
- **不隐藏含假设的行**：假设行必须对审阅者可见；不要用 `hidden="1"` 属性
- 在假设区顶部或专门单元格注明"最后更新"日期，记录模型最后修改时间
