# 中国高校论文模板指南（中国高校论文模板指南）

## 本指南存在的原因

中国高校论文模板（.docx）的结构模式与西方模板差异显著。假设西方惯例（Heading1/Heading2/Normal）的 agent 会反复失败。本指南记录了中国模板中的实际模式。

## 常见 StyleId 模式

### 模式 A：数字 ID（中文 Word 模板中最常见）

| 样式用途 | styleId | w:name | w:basedOn |
|--------------|---------|--------|-----------|
| 正文 Normal | `a` | "Normal" | — |
| 默认段落字体 | `a0` | "Default Paragraph Font" | — |
| 一级标题（章标题） | `1` | "heading 1" | `a` |
| 二级标题（节标题） | `2` | "heading 2" | `a` |
| 三级标题（小节标题） | `3` | "heading 3" | `a` |
| 目录 1 | `11` | "toc 1" | `a` |
| 目录 2 | `21` | "toc 2" | `a` |
| 目录 3 | `31` | "toc 3" | `a` |
| 页眉 | `a3` | "header" | `a` |
| 页脚 | `a4` | "footer" | `a` |
| 目录标题 | `10` | "TOC Heading" | `1` |

### 模式 B：英文 ID（较少见，通常来自国际模板）
标准 Heading1/Heading2/Heading3/Normal — 遵循西方模式。

### 模式 C：混合（部分中文，部分英文）
部分模板用中文名定义自定义样式：
| 样式用途 | styleId | w:name |
|--------------|---------|--------|
| 论文标题 | `lunwenbiaoti` | "论文标题" |
| 章标题 | `zhangbiaoti` | "章标题" |
| 正文 | `zhengwen` | "正文" |

### 如何识别是哪种模式

```bash
# 从模板提取所有 styleId
$CLI analyze --input template.docx --styles-only

# 或手动：
# unzip template.docx word/styles.xml
# 在解压的文件中搜索 w:styleId=
```

查看前几个 styleId。若看到 `1`、`2`、`3`、`a`、`a0` → 模式 A。
若看到 `Heading1`、`Normal` → 模式 B。

## 标准论文结构

中国高校论文遵循高度标准化的结构：

```
┌─────────────────────────────────────┐
│ 封面（Cover Page）                   │  ← 通常 1-2 页
│   - 校名、校徽                       │
│   - 论文题目（title）                 │
│   - 作者、导师、院系、日期             │
├─────────────────────────────────────┤
│ 学术诚信承诺书 / 独创性声明            │  ← 1 页
│   (Academic Integrity Declaration)   │
├─────────────────────────────────────┤
│ 中文摘要（Chinese Abstract）          │  ← 1-2 页
│   - "摘 要" 标题                     │
│   - 摘要正文                         │
│   - "关键词：" 行                    │
├─────────────────────────────────────┤
│ 英文摘要（English Abstract）          │  ← 1-2 页
│   - "ABSTRACT" 标题                  │
│   - 摘要正文                         │
│   - "Keywords:" 行                   │
├─────────────────────────────────────┤
│ 目录（Table of Contents）             │  ← 1-3 页
│   - 通常在 SDT 块内                  │
│   - 静态示例条目                      │
│   - 目录域代码                        │
├─────────────────────────────────────┤
│ 正文（Body）                          │  ← 主要内容
│   第1章 绪论                          │
│   1.1 研究背景                        │
│   1.2 研究目的和意义                   │
│   第2章 文献综述                       │
│   ...                                │
│   第N章 结论与展望                     │
├─────────────────────────────────────┤
│ 参考文献（References）                │  ← 样式不同
├─────────────────────────────────────┤
│ 致谢（Acknowledgments）              │  ← 可选
├─────────────────────────────────────┤
│ 附录（Appendices）                    │  ← 可选
└─────────────────────────────────────┘
```

## 识别模板中的区域边界

模板包含必须替换的示例内容。以下是查找各区域的方法：

### 区域 A（前置部分）— 保留自模板
- 起始：段落 0
- 结束：第一个章标题之前的那一段
- 包含：封面、声明、摘要、目录
- 如何检测结束：搜索第一个样式为 `1`（或 Heading1）且含"第1章"或"绪论"的段落

### 区域 B（正文内容）— 用用户内容替换
- 起始：第一个章标题（"第1章..."）
- 结束："参考文献"标题（含）或致谢前的最后一段正文
- 如何检测：
  ```python
  for i, el in enumerate(body_elements):
      text = get_text(el)
      style = get_style(el)
      if style in ('1', 'Heading1') and ('第1章' in text or '绪论' in text):
          zone_b_start = i
      if '参考文献' in text:
          zone_b_end = i
  ```

### 区域 C（后置部分）— 保留自模板（或移除）
- 起始于：参考文献之后
- 包含：致谢、附录、末尾 sectPr

## 中文论文模板的字体预期

| 元素 | 字体 | 字号 | 磅值（pt） | w:sz |
|---------|------|------------|-----------|------|
| 论文标题 | 华文中宋 or 黑体 | 二号 or 小二 | 22pt or 18pt | 44 or 36 |
| 章标题（H1） | 黑体 | 三号 | 16pt | 32 |
| 节标题（H2） | 黑体 | 四号 | 14pt | 28 |
| 小节标题（H3） | 黑体 | 小四 | 12pt | 24 |
| 正文 | 宋体 | 小四 | 12pt | 24 |
| 页眉 | 宋体 | 五号 | 10.5pt | 21 |
| 页脚/页码 | 宋体 | 五号 | 10.5pt | 21 |
| 表格内容 | 宋体 | 五号 | 10.5pt | 21 |
| 参考文献条目 | 宋体 | 五号 | 10.5pt | 21 |

## CJK 正文的 RunFonts

```xml
<w:rFonts w:ascii="Times New Roman" w:hAnsi="Times New Roman"
          w:eastAsia="宋体" w:cs="Times New Roman"/>
```

标题用：
```xml
<w:rFonts w:ascii="Times New Roman" w:hAnsi="Times New Roman"
          w:eastAsia="黑体" w:cs="Times New Roman"/>
```

重要：清理直接格式时，始终保留 w:eastAsia。
移除它会导致中文回退到错误字体。

## 中文模板的常见错误

1. **搜索 `Heading1`** — 中文模板用 `1`，不是 `Heading1`
2. **清除所有 rFonts** — 必须保留 eastAsia 字体声明
3. **假设"第1章"是第一段** — 它通常是封面/摘要/目录之后的第 100+ 段
4. **忽略目录中的 SDT 块** — 目录被 SDT 包装，不仅是域代码
5. **错误行距** — 中文论文通常用固定 20pt（line="400"）或 22pt（line="440"），而非政府公文的 28pt
6. **缺少分节符** — 每个区域（摘要、目录、正文）通常有各自的 sectPr 以实现不同的页眉/页脚

## 样式映射速查

源文档用西方 ID 而模板用中文数字 ID 时：

```json
{
  "Heading1": "1",
  "Heading2": "2",
  "Heading3": "3",
  "Heading4": "3",
  "Normal": "a",
  "BodyText": "a",
  "ListParagraph": "a",
  "Caption": "a",
  "TOC1": "11",
  "TOC2": "21",
  "TOC3": "31"
}
```

源用中文数字 ID 而模板用西方 ID 时 — 反向映射。
