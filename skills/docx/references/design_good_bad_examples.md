# 文档设计好坏对比 — 具体 OpenXML 示例

一份并排参考，展示常见设计错误及其修复，含精确 OpenXML 参数值。用于培养对"什么让文档看起来专业而非业余"的直觉。

格式：每个对比先展示**坏**版本（错误），再展示**好**版本（修复），附 OpenXML 标记和简短说明。

---

## 1. 字号灾难

### 1a. 无层级 — 一切同号

**坏：正文=12pt，H1=12pt 粗体**
```
┌──────────────────────────────────┐
│ 引言                             │  ← 12pt 粗体… 同样视觉重量
│ 这是报告的正文。                  │  ← 12pt 常规
│ 讨论季度评审的发现。              │
│ 方法论                            │  ← 章节从哪开始？
│ 我们从企业的三个来源              │
│ 收集了数据。                      │
└──────────────────────────────────┘
```
```xml
<!-- H1：粗体但与正文同号 — 无视觉分隔 -->
<w:rPr><w:b/><w:sz w:val="24"/></w:rPr>
<!-- 正文 -->
<w:rPr><w:sz w:val="24"/></w:rPr>
```

**好：模数比例 — 正文=11pt，H3=13pt，H2=16pt，H1=20pt**
```
┌──────────────────────────────────┐
│                                  │
│ 引言                             │  ← 20pt，明显是标题
│                                  │
│ 这是报告的正文。                  │  ← 11pt，舒适阅读字号
│ 讨论季度评审的发现。              │
│                                  │
│ 方法论                            │  ← 20pt，章节分隔明显
│                                  │
│ 我们从企业的三个来源              │
│ 收集了数据。                      │
└──────────────────────────────────┘
```
```xml
<!-- H1：20pt = w:sz 40 -->
<w:rPr><w:rFonts w:ascii="Calibri Light"/><w:sz w:val="40"/></w:rPr>
<!-- H2：16pt = w:sz 32 -->
<w:rPr><w:rFonts w:ascii="Calibri Light"/><w:sz w:val="32"/></w:rPr>
<!-- H3：13pt = w:sz 26，粗体 -->
<w:rPr><w:rFonts w:ascii="Calibri"/><w:b/><w:sz w:val="26"/></w:rPr>
<!-- 正文：11pt = w:sz 22 -->
<w:rPr><w:rFonts w:ascii="Calibri"/><w:sz w:val="22"/></w:rPr>
```
**为何更好：** 清晰的字号递进（每级约 1.25x）让读者无需读一个字即可瞬间识别结构。

---

### 1b. 对比过度 — 儿童读物感

**坏：H1=28pt 配正文=10pt（比率 2.8x）**
```
┌──────────────────────────────────┐
│                                  │
│ 季度报告                          │  ← 28pt，主导整页
│                                  │
│ 这是设得很小的正文                │  ← 10pt，费力阅读
│ 与标题的对比使它                  │
│ 感觉像海报。                      │
└──────────────────────────────────┘
```
```xml
<w:rPr><w:b/><w:sz w:val="56"/></w:rPr>  <!-- 28pt 标题 -->
<w:rPr><w:sz w:val="20"/></w:rPr>         <!-- 10pt 正文 -->
```

**好：H1=20pt 配正文=11pt（比率约 1.8x）**
```xml
<w:rPr><w:sz w:val="40"/></w:rPr>  <!-- 20pt 标题 -->
<w:rPr><w:sz w:val="22"/></w:rPr>  <!-- 11pt 正文 -->
```
**为何更好：** 标题与正文比率在 1.5x 到 2.0x 之间读起来像"有结构"而非"在喊叫"。

## 2. 间距罪行

### 2a. 文字墙 — 无段距或行距

**坏：单倍行距，段间 0pt**
```
┌──────────────────────────────────┐
│发现表明培训时长与绩效             │
│指标之间存在强相关性。             │
│进一步分析显示，预算               │  ← 无间隙 — 新段落从哪
│更高的部门在所有衡量类别           │     开始？
│中都取得了更好结果。               │
└──────────────────────────────────┘
```
```xml
<w:pPr>
  <w:spacing w:line="240" w:lineRule="auto"/>  <!-- 1.0 间距 (240/240) -->
  <w:spacing w:after="0"/>                     <!-- 无段距 -->
</w:pPr>
```

**好：1.15x 行距，每段后 8pt**
```
┌──────────────────────────────────┐
│发现表明培训时长与绩效             │
│指标之间存在强相关性。             │  ← 行间略多空气
│                                  │  ← 8pt 间隙标记新段落
│进一步分析显示，预算               │
│更高的部门在所有衡量类别           │
│中都取得了更好结果。               │
└──────────────────────────────────┘
```
```xml
<w:pPr>
  <w:spacing w:line="276" w:lineRule="auto"/>  <!-- 1.15x (276/240) -->
  <w:spacing w:after="160"/>                   <!-- 8pt = 160 twips -->
</w:pPr>
```
**为何更好：** 行距给每行呼吸空间；段距分隔意思而不浪费整空行。

---

### 2b. 漂浮标题 — 上下间距相同

**坏：标题前 12pt 后 12pt**
```
┌──────────────────────────────────┐
│ ...上一节结束。                  │
│                                  │  ← 12pt 间隙
│ 第二节                           │  ← 标题漂浮在中间
│                                  │  ← 12pt 间隙
│ 第二节内容开始。                  │
└──────────────────────────────────┘
```
```xml
<w:pPr>
  <w:spacing w:before="240" w:after="240"/>  <!-- 两边 12pt -->
</w:pPr>
```

**好：标题前 24pt 后 8pt**
```
┌──────────────────────────────────┐
│ ...上一节结束。                  │
│                                  │
│                                  │  ← 24pt 间隙 — 清晰分节
│ 第二节                           │  ← 标题靠近其内容
│                                  │  ← 8pt 间隙
│ 第二节内容开始。                  │
└──────────────────────────────────┘
```
```xml
<w:pPr>
  <w:spacing w:before="480" w:after="160"/>  <!-- 前 24pt，后 8pt -->
</w:pPr>
```
**为何更好：** 邻近原则：标题属于其后的文本，所以上方多空间、下方少空间将其锚定到内容。

---

### 2c. 浪费间隙 — 到处巨大间距

**坏：每段后 24pt，包括正文**
```
┌──────────────────────────────────┐
│ 第一段文本在此。                  │
│                                  │
│                                  │  ← 每段后 24pt 间隙
│                                  │
│ 第二段文本在此。                  │
│                                  │
│                                  │
│                                  │
│ 第三段。                          │  ← 文档看起来大多是留白
└──────────────────────────────────┘
```
```xml
<w:spacing w:after="480"/>  <!-- 每段后 24pt = 480 twips -->
```

**好：成比例间距 — 正文=8pt，H2 后=6pt，H1 后=10pt**
```xml
<!-- 正文段落 -->
<w:spacing w:after="160"/>   <!-- 正文后 8pt -->
<!-- H1 -->
<w:spacing w:before="480" w:after="200"/>  <!-- 前 24pt，后 10pt -->
<!-- H2 -->
<w:spacing w:before="320" w:after="120"/>  <!-- 前 16pt，后 6pt -->
```
**为何更好：** 间距应按元素角色变化，创造视觉节奏而非统一间隙。

## 3. 页边距错误

### 3a. 拥挤页边距 — 文本贴到边缘

**坏：四边 0.5in**
```
┌────────────────────────────────────────────────┐
│文本几乎从纸边开始，一路横跨，形成极长的行，    │
│难以从行尾追踪回行首。眼睛每次换行都丢失位置。 │
└────────────────────────────────────────────────┘
```
```xml
<w:pgMar w:top="720" w:right="720" w:bottom="720" w:left="720"/>
<!-- 720 twips = 0.5in — Letter 纸上行长约 7.5in -->
```

**好：1in 页边距（标准）**
```xml
<w:pgMar w:top="1440" w:right="1440" w:bottom="1440" w:left="1440"/>
<!-- 1440 twips = 1.0in — 行长约 6.5in，对 11pt 正文理想 -->
```
**为何更好：** 最佳行长 60-75 字符。11pt Calibri 下，6.5in 宽度约每行 70 字符。

---

### 3b. 过度填充页边距 — 看起来像内容在躲藏

**坏：短文档用 2in 页边距**
```xml
<w:pgMar w:top="2880" w:right="2880" w:bottom="2880" w:left="2880"/>
<!-- 2880 twips = 2.0in — 仅 4.5in 文本宽，看起来填充过度 -->
```

**好：1in 标准，或正式文档用 1.25in**
```xml
<!-- 标准 -->
<w:pgMar w:top="1440" w:right="1440" w:bottom="1440" w:left="1440"/>
<!-- 正式/装订文档带装订线 -->
<w:pgMar w:top="1440" w:right="1440" w:bottom="1440" w:left="1800" w:gutter="0"/>
<!-- 1800 twips = 1.25in 左侧装订线 -->
```
**为何更好：** 页边距应为内容取景，而非压倒内容。1-1.25in 适用于几乎所有商业和学术文档。

## 4. 表格丑陋

### 4a. 监狱网格 — 每个单元格全边框

**坏：每个单元格四边 1pt 边框**
```
┌───────┬───────┬───────┬───────┐
│ 姓名  │ 部门  │ 分数  │ 等级  │
├───────┼───────┼───────┼───────┤
│ 张三  │ 工程  │ 92    │ A     │
├───────┼───────┼───────┼───────┤
│ 李四  │ 销售  │ 85    │ B     │
├───────┼───────┼───────┼───────┤
│ 王五  │ 工程  │ 78    │ C+    │
└───────┴───────┴───────┴───────┘
```
```xml
<w:tcBorders>
  <w:top w:val="single" w:sz="4" w:color="000000"/>
  <w:left w:val="single" w:sz="4" w:color="000000"/>
  <w:bottom w:val="single" w:sz="4" w:color="000000"/>
  <w:right w:val="single" w:sz="4" w:color="000000"/>
</w:tcBorders>
```

**好：三线表 — 顶粗、表头下中、表底粗**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  ← 1.5pt 顶边框
  姓名    部门    分数    等级
──────────────────────────────────  ← 0.75pt 表头分隔
  张三    工程    92      A
  李四    销售    85      B
  王五    工程    78      C+
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  ← 1.5pt 底边框
```
```xml
<!-- 表头行单元格顶边框 -->
<w:top w:val="single" w:sz="12" w:color="000000"/>    <!-- 1.5pt -->
<w:left w:val="nil"/><w:right w:val="nil"/>
<w:bottom w:val="single" w:sz="6" w:color="000000"/>  <!-- 0.75pt -->

<!-- 数据行单元格：无左/右/顶边框 -->
<w:top w:val="nil"/><w:left w:val="nil"/><w:right w:val="nil"/>
<w:bottom w:val="nil"/>

<!-- 末行底边框 -->
<w:bottom w:val="single" w:sz="12" w:color="000000"/> <!-- 1.5pt -->
```
**为何更好：** 移除内部边框让眼睛自由扫描数据。三条线提供结构而无视觉杂乱。

---

### 4b. 文本贴边框 — 无单元格内边距

**坏：零单元格边距**
```
┌──────────┬──────────┐
│姓名      │部门      │  ← 文本紧贴边框
├──────────┼──────────┤
│张三      │工程部    │
└──────────┴──────────┘
```
```xml
<w:tcMar>
  <w:top w:w="0" w:type="dxa"/>
  <w:start w:w="0" w:type="dxa"/>
  <w:bottom w:w="0" w:type="dxa"/>
  <w:end w:w="0" w:type="dxa"/>
</w:tcMar>
```

**好：垂直 0.08in，水平 0.12in 内边距**
```xml
<w:tcMar>
  <w:top w:w="115" w:type="dxa"/>      <!-- ~0.08in = 115 twips -->
  <w:start w:w="173" w:type="dxa"/>    <!-- ~0.12in = 173 twips -->
  <w:bottom w:w="115" w:type="dxa"/>
  <w:end w:w="173" w:type="dxa"/>
</w:tcMar>
```
**为何更好：** 内边距给单元格内文本呼吸空间，使每个值更易读。

---

### 4c. 隐形表头 — 表头行与数据同样式

**坏：表头行与数据无法区分**
```xml
<!-- 表头单元格 run 属性 — 与数据相同 -->
<w:rPr><w:sz w:val="22"/></w:rPr>
```

**好：粗体表头文本、微妙背景填充、底边框**
```xml
<!-- 表头单元格 run 属性 -->
<w:rPr><w:b/><w:sz w:val="22"/><w:color w:val="333333"/></w:rPr>

<!-- 表头单元格底纹 -->
<w:tcPr>
  <w:shd w:val="clear" w:color="auto" w:fill="F2F2F2"/>  <!-- 浅灰背景 -->
  <w:tcBorders>
    <w:bottom w:val="single" w:sz="8" w:color="666666"/>  <!-- 1pt 分隔 -->
  </w:tcBorders>
</w:tcPr>

<!-- 标记行为表头（分页时重复） -->
<w:trPr><w:tblHeader/></w:trPr>
```
**为何更好：** 鲜明的表头样式让读者瞬间定位列含义，尤其在跨页长表中。`w:tblHeader` 元素确保表头行在每页重复。

## 5. 字体配对失败

### 5a. 视觉混乱 — 字体过多

**坏：一份文档 4+ 种字体**
```xml
<!-- H1 用 Impact -->
<w:rPr><w:rFonts w:ascii="Impact"/><w:sz w:val="40"/></w:rPr>
<!-- H2 用 Georgia -->
<w:rPr><w:rFonts w:ascii="Georgia"/><w:sz w:val="32"/></w:rPr>
<!-- 正文用 Verdana -->
<w:rPr><w:rFonts w:ascii="Verdana"/><w:sz w:val="22"/></w:rPr>
<!-- 题注用 Courier New -->
<w:rPr><w:rFonts w:ascii="Courier New"/><w:sz w:val="18"/></w:rPr>
```

**好：一个字族配字重变化，或两个互补字族**
```xml
<!-- H1：Calibri Light（Calibri 字族的细字重） -->
<w:rPr><w:rFonts w:ascii="Calibri Light"/><w:sz w:val="40"/></w:rPr>
<!-- H2：Calibri Light -->
<w:rPr><w:rFonts w:ascii="Calibri Light"/><w:sz w:val="32"/></w:rPr>
<!-- 正文：Calibri（常规字重） -->
<w:rPr><w:rFonts w:ascii="Calibri"/><w:sz w:val="22"/></w:rPr>
<!-- 题注：Calibri -->
<w:rPr><w:rFonts w:ascii="Calibri"/><w:sz w:val="18"/></w:rPr>
```
**为何更好：** 限制为一或两个字族创造视觉连贯。用字号和字重变化，而非字体变化。

---

### 5b. 性格不匹配 — Comic Sans 遇上 Times New Roman

**坏：**
```xml
<w:rPr><w:rFonts w:ascii="Comic Sans MS"/><w:sz w:val="36"/></w:rPr>  <!-- 标题 -->
<w:rPr><w:rFonts w:ascii="Times New Roman"/><w:sz w:val="24"/></w:rPr> <!-- 正文 -->
```

**好：性格兼容的字体**
```xml
<w:rPr><w:rFonts w:ascii="Calibri Light"/><w:sz w:val="36"/></w:rPr>   <!-- 标题 -->
<w:rPr><w:rFonts w:ascii="Calibri"/><w:sz w:val="22"/></w:rPr>          <!-- 正文 -->
```
**为何更好：** 配对的字体应共享相似的正式程度和几何性格。Comic Sans 俏皮/非正式；Times New Roman 正式/传统。它们冲突。

---

### 5c. 一切粗体 — 什么都不突出

**坏：正文、标题、题注、一切都粗体**
```xml
<w:rPr><w:b/><w:sz w:val="40"/></w:rPr>  <!-- 标题：粗体 -->
<w:rPr><w:b/><w:sz w:val="22"/></w:rPr>  <!-- 正文：也粗体 -->
<w:rPr><w:b/><w:sz w:val="18"/></w:rPr>  <!-- 题注：仍粗体 -->
```

**好：粗体仅用于标题和关键术语**
```xml
<w:rPr><w:b/><w:sz w:val="40"/></w:rPr>   <!-- H1：粗体 -->
<w:rPr><w:sz w:val="32"/></w:rPr>          <!-- H2：仅字号即可 -->
<w:rPr><w:sz w:val="22"/></w:rPr>          <!-- 正文：常规字重 -->
<w:rPr><w:b/><w:sz w:val="22"/></w:rPr>    <!-- 内联关键术语：粗体 -->
<w:rPr><w:sz w:val="18"/></w:rPr>          <!-- 题注：常规，小 -->
```
**为何更好：** 一切都强调时，什么都没强调。粗体应是信号，而非默认。

---

## 6. 颜色滥用

### 6a. 彩虹标题

**坏：每个标题级一种不同亮色**
```xml
<w:rPr><w:color w:val="FF0000"/><w:sz w:val="40"/></w:rPr>  <!-- H1：红 -->
<w:rPr><w:color w:val="00AA00"/><w:sz w:val="32"/></w:rPr>  <!-- H2：绿 -->
<w:rPr><w:color w:val="0000FF"/><w:sz w:val="26"/></w:rPr>  <!-- H3：蓝 -->
```

**好：标题单一强调色，正文黑或深灰**
```xml
<!-- 所有标题用同一柔和强调色 -->
<w:rPr><w:color w:val="1F4E79"/><w:sz w:val="40"/></w:rPr>  <!-- H1：深蓝 -->
<w:rPr><w:color w:val="1F4E79"/><w:sz w:val="32"/></w:rPr>  <!-- H2：同蓝 -->
<w:rPr><w:color w:val="1F4E79"/><w:sz w:val="26"/></w:rPr>  <!-- H3：同蓝 -->
<!-- 正文近黑 -->
<w:rPr><w:color w:val="333333"/><w:sz w:val="22"/></w:rPr>
```
**为何更好：** 单一强调色建立品牌一致性。多种亮色争夺注意力，显得不专业。

---

### 6b. 低对比 — 白底浅灰

**坏：白底 #CCCCCC 文本**
```xml
<w:rPr><w:color w:val="CCCCCC"/></w:rPr>
<!-- 对比度：约 1.6:1 — 不达 WCAG AA（最低 4.5:1） -->
```

**好：白底 #333333 文本**
```xml
<w:rPr><w:color w:val="333333"/></w:rPr>
<!-- 对比度：约 12:1 — 达 WCAG AAA -->
```
**为何更好：** 足够对比不仅是无障碍要求；它使文本对每个人都更易读，尤其在打印文档中。

---

### 6c. 亮色正文

**坏：饱和色正文**
```xml
<w:rPr><w:color w:val="0066FF"/><w:sz w:val="22"/></w:rPr>  <!-- 蓝色正文 -->
```

**好：颜色仅用于标题和内联强调**
```xml
<!-- 正文：中性深色 -->
<w:rPr><w:color w:val="333333"/><w:sz w:val="22"/></w:rPr>
<!-- 超链接：此处颜色是功能性 -->
<w:rPr><w:color w:val="0563C1"/><w:u w:val="single"/></w:rPr>
```
**为何更好：** 彩色正文在长阅读中导致眼疲劳。颜色留给需吸引注意的元素（标题、链接、警告）。

---

## 7. 列表格式问题

### 7a. 项目符号贴页边距 — 无缩进

**坏：列表项从左边距开始**
```
┌──────────────────────────────────┐
│这是一段文本。                    │
│• 第一项                          │  ← 符号在页边距，无缩进
│• 第二项                          │
│• 第三项                          │
│下一段在此继续。                  │
└──────────────────────────────────┘
```
```xml
<w:pPr>
  <w:ind w:left="0" w:hanging="0"/>
</w:pPr>
```

**好：0.25in 左缩进，项目符号悬挂缩进**
```
┌──────────────────────────────────┐
│这是一段文本。                    │
│   • 第一项                       │  ← 缩进，明显是列表
│   • 第二项                       │
│   • 第三项                       │
│下一段在此继续。                  │
└──────────────────────────────────┘
```
```xml
<w:pPr>
  <w:ind w:left="360" w:hanging="360"/>  <!-- 0.25in = 360 twips -->
  <w:numPr>
    <w:ilvl w:val="0"/>
    <w:numId w:val="1"/>
  </w:numPr>
</w:pPr>
```
嵌套列表每级递增 360 twips：
```xml
<!-- 第 1 级 -->
<w:ind w:left="720" w:hanging="360"/>   <!-- 左 0.5in -->
<!-- 第 2 级 -->
<w:ind w:left="1080" w:hanging="360"/>  <!-- 左 0.75in -->
```
**为何更好：** 缩进视觉上将列表与正文分开，使嵌套级别清晰。

---

### 7b. 列表项用完整段距

**坏：列表项与正文一样 8-10pt 间距**
```
┌──────────────────────────────────┐
│   • 第一项                       │
│                                  │  ← 10pt 间隙 — 看起来像独立
│   • 第二项                       │     段落，而非列表
│                                  │
│   • 第三项                       │
└──────────────────────────────────┘
```
```xml
<w:spacing w:after="200"/>  <!-- 每个列表项后 10pt -->
```

**好：列表项间紧密间距（2-4pt）**
```
┌──────────────────────────────────┐
│   • 第一项                       │
│   • 第二项                       │  ← 2pt 间隙 — 内聚列表
│   • 第三项                       │
└──────────────────────────────────┘
```
```xml
<w:spacing w:after="40" w:line="276" w:lineRule="auto"/>  <!-- 后 2pt -->
<!-- 或 4pt： -->
<w:spacing w:after="80"/>
```
**为何更好：** 紧密间距将列表项归为单一单元，符合读者对列表行为的期望。

---

## 8. 页眉/页脚问题

### 8a. 页眉文本过大 — 与正文争抢

**坏：页眉 12pt，与正文同**
```
┌──────────────────────────────────┐
│ 季度报告 - 2025 第三季度         │  ← 12pt 页眉，与正文同
│──────────────────────────────────│
│ 引言                             │
│ 这是正文…                        │  ← 12pt 正文 — 页眉分散注意
└──────────────────────────────────┘
```
```xml
<!-- 页眉段落 -->
<w:rPr><w:sz w:val="24"/></w:rPr>  <!-- 12pt，与正文同 -->
```

**好：页眉 9pt，灰色，微妙**
```
┌──────────────────────────────────┐
│ 季度报告 - 2025 第三季度         │  ← 9pt，灰 — 在场但安静
│──────────────────────────────────│
│ 引言                             │
│ 这是正文…                        │  ← 正文作为主体突出
└──────────────────────────────────┘
```
```xml
<!-- 页眉段落 -->
<w:rPr>
  <w:sz w:val="18"/>                <!-- 9pt -->
  <w:color w:val="808080"/>         <!-- 中灰 -->
</w:rPr>
<w:pPr>
  <w:pBdr>
    <w:bottom w:val="single" w:sz="4" w:color="D9D9D9"/>  <!-- 微妙分隔 -->
  </w:pBdr>
</w:pPr>
```
**为何更好：** 页眉是参考信息，非主要内容。应可读但视觉上从属。

---

### 8b. 长文档无页码

**坏：20 页文档无页码**
```xml
<!-- 页脚节：空或缺失 -->
```

**好：页脚页码，右对齐或居中**
```xml
<!-- 带页码域的页脚段落 -->
<w:p>
  <w:pPr>
    <w:jc w:val="center"/>
    <w:rPr><w:sz w:val="18"/><w:color w:val="808080"/></w:rPr>
  </w:pPr>
  <w:r>
    <w:rPr><w:sz w:val="18"/><w:color w:val="808080"/></w:rPr>
    <w:fldChar w:fldCharType="begin"/>
  </w:r>
  <w:r>
    <w:instrText> PAGE </w:instrText>
  </w:r>
  <w:r>
    <w:fldChar w:fldCharType="separate"/>
  </w:r>
  <w:r>
    <w:t>1</w:t>
  </w:r>
  <w:r>
    <w:fldChar w:fldCharType="end"/>
  </w:r>
</w:p>
```
**为何更好：** 页码对超过约 3 页的任何文档的导航至关重要。读者需引用特定页，打印文档需排序机制。

## 9. CJK 特有错误

### 9a. 用斜体做中文强调

**坏：斜体应用于中文文本**
```xml
<w:rPr>
  <w:i/>
  <w:rFonts w:eastAsia="SimSun"/>
  <w:sz w:val="24"/>
</w:rPr>
```
CJK 字形无真正斜体形式。渲染器施加合成倾斜，看起来破损丑陋 — 字符显得别扭地倾斜。

**好：用粗体或着重号做中文强调**
```xml
<!-- 选项 A：粗体强调 -->
<w:rPr>
  <w:b/>
  <w:rFonts w:eastAsia="SimHei"/>  <!-- 切换到支持粗体的字体 -->
  <w:sz w:val="24"/>
</w:rPr>

<!-- 选项 B：着重号（字符下方圆点） -->
<w:rPr>
  <w:em w:val="dot"/>
  <w:rFonts w:eastAsia="SimSun"/>
  <w:sz w:val="24"/>
</w:rPr>
```
**为何更好：** 中文排版有自己的强调传统。粗体和着重号是原生 CJK 惯例；斜体是拉丁文字概念，无法移植。

---

### 9b. 中文字符用拉丁字体

**坏：仅设 ASCII 字体，无 EastAsia 字体**
```xml
<w:rPr>
  <w:rFonts w:ascii="Arial"/>  <!-- 无 eastAsia 属性 -->
  <w:sz w:val="24"/>
</w:rPr>
<!-- Word 回退到随机字体。中文字符可能以错误度量、
     不一致笔画宽度或缺失字形渲染。 -->
```

**好：ASCII 字体旁显式 EastAsia 字体**
```xml
<w:rPr>
  <w:rFonts w:ascii="Calibri" w:hAnsi="Calibri" w:eastAsia="Microsoft YaHei"/>
  <w:sz w:val="22"/>
</w:rPr>
```
正式/学术中文文档：
```xml
<w:rPr>
  <w:rFonts w:ascii="Times New Roman" w:hAnsi="Times New Roman"
            w:eastAsia="SimSun"/>
  <w:sz w:val="24"/>  <!-- 小四 12pt -->
</w:rPr>
```
**为何更好：** 设置 `w:eastAsia` 确保中文字符用为 CJK 字形设计的字体渲染，笔画宽度、间距、度量正确。

---

### 9c. 密集 CJK 文本用英文行距

**坏：中文正文用 1.15x 行距**
```xml
<w:spacing w:line="276" w:lineRule="auto"/>  <!-- 1.15x — 对 CJK 太紧 -->
```
CJK 字符比拉丁字母更高更密。1.15x 下，中文文本行感觉拥挤难读。

**好：CJK 正文 12pt（小四）用 1.5x 行距或固定 28pt**
```xml
<!-- 选项 A：1.5x 比例 -->
<w:spacing w:line="360" w:lineRule="auto"/>  <!-- 360/240 = 1.5x -->

<!-- 选项 B：固定 28pt（小四/12pt CJK 正文标准） -->
<w:spacing w:line="560" w:lineRule="exact"/>  <!-- 28pt = 560 twips -->
```
公文（政府文档）三号/16pt 正文：
```xml
<w:spacing w:line="580" w:lineRule="exact"/>  <!-- 固定 29pt 行距 -->
```
**为何更好：** CJK 字符占据完整 em 方格，无升部/降部提供自然间隙。额外行距补偿，改善密集文本块的可读性。

---

## 10. 整体文档质感

### 学生作业 vs 专业文档

**坏："学生作业" — 每个设置都是 Word 默认，无有意选择**
```xml
<!-- 一切默认：Calibri 11pt，无标题样式，1.08 间距 -->
<w:rPr><w:rFonts w:ascii="Calibri"/><w:sz w:val="22"/></w:rPr>
<w:pPr><w:spacing w:after="160" w:line="259" w:lineRule="auto"/></w:pPr>
<!-- 标题：仅粗体正文，无样式应用 -->
<w:rPr><w:b/><w:sz w:val="22"/></w:rPr>
<!-- 无分节符，无页眉/页脚，无页码 -->
<!-- 表格用默认全网格边框 -->
<!-- 无有意的颜色或间距变化 -->
```

**好：每个层级的刻意设计**
```xml
<!-- 定义主题字体 -->
<w:rFonts w:asciiTheme="minorHAnsi" w:hAnsiTheme="minorHAnsi"/>

<!-- H1：Calibri Light 20pt，深蓝，慷慨间距 -->
<w:pPr>
  <w:pStyle w:val="Heading1"/>
  <w:spacing w:before="480" w:after="200"/>
</w:pPr>
<w:rPr>
  <w:rFonts w:ascii="Calibri Light"/>
  <w:color w:val="1F4E79"/>
  <w:sz w:val="40"/>
</w:rPr>

<!-- H2：Calibri Light 16pt，同蓝 -->
<w:pPr>
  <w:pStyle w:val="Heading2"/>
  <w:spacing w:before="320" w:after="120"/>
</w:pPr>
<w:rPr>
  <w:rFonts w:ascii="Calibri Light"/>
  <w:color w:val="1F4E79"/>
  <w:sz w:val="32"/>
</w:rPr>

<!-- 正文：Calibri 11pt，深灰，1.15 间距，后 8pt -->
<w:pPr>
  <w:spacing w:after="160" w:line="276" w:lineRule="auto"/>
</w:pPr>
<w:rPr>
  <w:rFonts w:ascii="Calibri"/>
  <w:color w:val="333333"/>
  <w:sz w:val="22"/>
</w:rPr>

<!-- 表格：三线样式，带内边距单元格，重复表头 -->
<!-- 页眉/页脚：9pt 灰带页码 -->
<!-- 页边距：四边 1in -->
<w:pgMar w:top="1440" w:right="1440" w:bottom="1440" w:left="1440"/>
```
**为何更好：** 专业文档源于跨所有设计维度的刻意、一致选择。每个元素强化同一视觉语言。读者可能不会自觉地注意到好排版，但能感受到可信度和可读性的差异。
