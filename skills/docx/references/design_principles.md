## 1. 留白与呼吸空间

### 为何有效

人眼并非连续阅读。它以扫视（saccades）跳动，固定在小的词簇上。留白为这些注视点提供着陆区，并给读者的周边视觉一个"画框"，使每个文本块感觉易于处理。当页面被塞满到边缘时，每次瞥见返回的文本超过工作记忆能缓冲的量，引发疲劳和回避。

关于内容密度的研究一致表明：

- **60-70% 内容覆盖率**感觉舒适且专业。
- **80%+** 开始感觉密集、官僚。
- **90%+** 感觉压抑 — 读者不自觉地赶读或跳读。
- **低于 50%** 感觉浪费或做作（除非有意为之，如诗歌）。

更宽的页边距还承载文化信号。学术和奢华文档使用慷慨的页边距（1.25-1.5 英寸）。内部备忘录和草稿使用较窄的页边距（0.75-1.0 英寸）。页边距宽度在读者读到一个字之前就告诉了他们文档投入了多少心思。

行距有直接的生理基础：每次换行后，眼睛必须追踪回到下一行开头。若行距太近，眼睛会"滑"到错误的行。若太远，眼睛失去连续感。最佳点是字号的 120-145%。

**经验法则：拿不准时，多留空间，而非少留。**

### 好的示例

```
商业文档四边页边距：1 英寸（1440 twips）。
行距：1.15（240 twips 每行下 276 twips = 115%）。
正文段后距：8pt（160 twips）。
```

```xml
<!-- 页边距：四边 1 英寸 = 1440 twips -->
<w:pgMar w:top="1440" w:right="1440" w:bottom="1440" w:left="1440"
         w:header="720" w:footer="720" w:gutter="0"/>

<!-- 正文段落：1.15 行距，段后 8pt -->
<w:pPr>
  <w:spacing w:after="160" w:line="276" w:lineRule="auto"/>
</w:pPr>
```

这产生的页面内容约占 65% 面积。读者看到清晰的上下呼吸空间，段落分明又不显得割裂。

```
  页面布局（好）：
  +----------------------------------+
  |           1" 页边距               |
  |   +------------------------+    |
  |   | 标题                   |    |
  |   |                        |    |
  |   | 正文在此，行间有        |    |
  |   | 舒适的间距。            |    |
  |   |                        |    |
  |   |                        |    |  <- 段落间可见间隙
  |   | 另一段正文紧随其后。     |    |
  |   |                        |    |
  |   +------------------------+    |
  |           1" 页边距               |
  +----------------------------------+
```

### 坏的示例

```xml
<!-- 拥挤的页边距：0.5 英寸 = 720 twips -->
<w:pgMar w:top="720" w:right="720" w:bottom="720" w:left="720"
         w:header="360" w:footer="360" w:gutter="0"/>

<!-- 无段间距，单倍行距 -->
<w:pPr>
  <w:spacing w:after="0" w:line="240" w:lineRule="auto"/>
</w:pPr>
```

这填满约 85% 页面。文本从边到边，无视觉休息点。读者看到一堵文字墙。

```
  页面布局（坏）：
  +----------------------------------+
  | 标题                              |
  | 正文紧贴页边距，行间和段间        |
  | 没有任何间距。另一段从这里        |
  | 开始，读者分不清一个意思在哪      |
  | 结束、另一个从哪开始，因为一切    |
  | 都模糊成一个密集的文字块。        |
  +----------------------------------+
```

### 快速测试

1. 在文档查看器中缩放到 50%。若你看不到文本块之间清晰的白色"通道"，间距太紧。
2. 打印测试页。一臂距离持看。文本区应像悬浮于白色中的矩形，而非填满整页。
3. 检查：正文行距值是否至少 264（1.1x 的 `w:line`）？若是 240（单倍），对超过 10pt 的任何内容都太紧。

## 2. 对比与比例

### 为何有效

大脑通过相对差异而非绝对大小处理视觉层级。11pt 正文上方的 20pt 标题产生清晰的"这很重要"信号。但若每个标题都是 20pt、每个副标题都是 19pt，大脑无法区分 — 它们合并为同一级。

关键洞见是**模数比例**：按一致比率增长的字号。这映射自然比例，之所以和谐，与音乐音程同理。

常见比例及其性格：

| 比率 | 名称           | 性格                       | 示例进程（从 11pt） |
|-------|----------------|---------------------------------|---------------------------------|
| 1.200 | 小三度    | 微妙、精致                 | 11 → 13.2 → 15.8 → 19.0       |
| 1.250 | 大三度    | 平衡、专业          | 11 → 13.75 → 17.2 → 21.5      |
| 1.333 | 纯四度 | 强烈、权威           | 11 → 14.7 → 19.5 → 26.0       |
| 1.414 | 增四度  | 戏剧、演示风格    | 11 → 15.6 → 22.0 → 31.1       |

对大多数商业文档，1.25（大三度）效果最佳：

```
正文  = 11pt  (w:sz="22")
H3    = 13pt  (w:sz="26")   -- 11 * 1.25 ≈ 13.75，取整 13
H2    = 16pt  (w:sz="32")   -- 13 * 1.25 ≈ 16.25，取整 16
H1    = 20pt  (w:sz="40")   -- 16 * 1.25 = 20
```

除字号外，**字重对比**在不消耗垂直空间的情况下创造层级。Regular（400）vs Bold（700）在任何尺寸下都可见。Semi-bold（600）vs Regular 很微妙，最好避免，除非同时变化字号或颜色。

**颜色对比**增加第三维度。深蓝标题（#1F3864）配较柔和的深灰正文（#333333）无需巨大字号跳跃即可暗示"标题"。纯黑（#000000）正文在白底上比必要的更刺眼 — #333333 或 #2D2D2D 降低眩光又不失可读性。

### 好的示例

```xml
<!-- H1：20pt，粗体，深海军蓝 -->
<w:rPr>
  <w:b/>
  <w:sz w:val="40"/>
  <w:color w:val="1F3864"/>
</w:rPr>

<!-- H2：16pt，粗体，深海军蓝 -->
<w:rPr>
  <w:b/>
  <w:sz w:val="32"/>
  <w:color w:val="1F3864"/>
</w:rPr>

<!-- H3：13pt，粗体，深海军蓝 -->
<w:rPr>
  <w:b/>
  <w:sz w:val="26"/>
  <w:color w:val="1F3864"/>
</w:rPr>

<!-- 正文：11pt，常规，深灰 -->
<w:rPr>
  <w:sz w:val="22"/>
  <w:color w:val="333333"/>
</w:rPr>
```

```
  视觉层级（好）：

  [████████████████████]        <- H1：20pt 粗体海军蓝（明显主导）
                                   （慷慨间距）
  [██████████████]              <- H2：16pt 粗体海军蓝（明显低一级）
                                   （适中间距）
  [████████████]                <- H3：13pt 粗体海军蓝（更小但仍粗体）
  [░░░░░░░░░░░░░░░░░░░░░░]    <- 正文：11pt 常规灰
  [░░░░░░░░░░░░░░░░░░░░░░]
  [░░░░░░░░░░░░░░░░░░░░░░]
```

每级与相邻级视觉上分明。即便在周边视觉中也能识别层级。

### 坏的示例

```xml
<!-- H1：14pt 粗体黑 -->
<w:rPr>
  <w:b/>
  <w:sz w:val="28"/>
  <w:color w:val="000000"/>
</w:rPr>

<!-- H2：13pt 粗体黑 -->
<w:rPr>
  <w:b/>
  <w:sz w:val="26"/>
  <w:color w:val="000000"/>
</w:rPr>

<!-- H3：12pt 粗体黑 -->
<w:rPr>
  <w:b/>
  <w:sz w:val="24"/>
  <w:color w:val="000000"/>
</w:rPr>

<!-- 正文：12pt 常规黑 -->
<w:rPr>
  <w:sz w:val="24"/>
  <w:color w:val="000000"/>
</w:rPr>
```

问题：
- H3（12pt 粗体）与正文（12pt 常规）仅字重不同 — 太微妙。
- H1（14pt）到 H2（13pt）是 1pt 步进 — 阅读距离下不可见。
- 一切都是纯黑，颜色不提供区分信号。
- 各级间比率约 1.07，过于扁平。

### 快速测试

1. **眯眼测试**：模糊视线或远离屏幕。你能数出标题层级数吗？若两级合并，对比不足。
2. **比率检查**：每个标题字号除以下一级字号。若任何比率低于 1.15，两级会看起来太相似。
3. **颜色检查**：扫视页面时，标题与正文是否看起来分明？若一切同色，你仅靠字号/字重，这把层级限制在约 3 个有效级别。

## 3. 邻近与分组

### 为何有效

格式塔邻近原则：靠得近的项目被感知为同组。在文档排版中，这意味着标题必须**更靠近它所引入的内容**，而非上方内容。

若标题等距于两段之间，它看起来孤立 — 读者眼睛不知道它属于上方还是下方文本。修复是非对称间距：**标题前大间距，后小间距**。

推荐比率是 2:1 或 3:1（段前 : 段后）。

同一原则适用于：
- **列表项**：项间距应小于段间距。列表中的项是一组，应视觉聚簇。
- **题注**：图注应靠近其图，而非漂浮在图与下一段之间。
- **表格标题**：标题紧贴表格上方，与上方文本有更多间距分隔。

### 好的示例

```xml
<!-- H2：前 18pt，后 6pt（3:1 比率） -->
<w:pPr>
  <w:pStyle w:val="Heading2"/>
  <w:spacing w:before="360" w:after="120"/>
</w:pPr>

<!-- 正文段落：前 0pt，后 8pt -->
<w:pPr>
  <w:spacing w:before="0" w:after="160"/>
</w:pPr>

<!-- 列表项：前 0pt，后 2pt（紧密分组） -->
<w:pPr>
  <w:pStyle w:val="ListParagraph"/>
  <w:spacing w:before="0" w:after="40"/>
</w:pPr>
```

```
  邻近（好）：

  ...上一节文本结束。
                                        <- 18pt 间隙 (w:before="360")
  ## 章节标题
                                        <- 6pt 间隙 (w:after="120")
  新节第一段
  内容在此继续。
                                        <- 8pt 间隙 (w:after="160")
  第二段紧随。

  标题明显"归属于"下方文本。
```

```
  列表分组（好）：

  考虑这些因素：
    - 第一项                        <- 项间 2pt 间隙
    - 第二项                        <- 项作为一组聚簇
    - 第三项
                                        <- 列表后 8pt 间隙
  下一段从这里开始。
```

### 坏的示例

```xml
<!-- H2：前 12pt，后 12pt（1:1 比率 — 孤立标题） -->
<w:pPr>
  <w:pStyle w:val="Heading2"/>
  <w:spacing w:before="240" w:after="240"/>
</w:pPr>

<!-- 列表项：与正文同间距（后 10pt） -->
<w:pPr>
  <w:pStyle w:val="ListParagraph"/>
  <w:spacing w:before="0" w:after="200"/>
</w:pPr>
```

```
  邻近（坏）：

  ...上一节文本结束。
                                        <- 12pt 间隙
  ## 章节标题
                                        <- 12pt 间隙（相同！）
  新节第一段。

  标题漂浮在节之间。不清楚它属于什么。
```

```
  列表分组（坏）：

  考虑这些因素：
                                        <- 10pt 间隙
    - 第一项
                                        <- 10pt 间隙（与段落相同）
    - 第二项
                                        <- 10pt 间隙
    - 第三项
                                        <- 10pt 间隙
  下一段。

  列表不像一组。每项看起来像
  碰巧带项目符号的独立段落。
```

### 快速测试

1. **遮挡测试**：遮挡标题文本。仅看留白，你能说出标题属于哪个文本块吗？若上下间隙相等，答案是"否"。
2. **数字检查**：标题的 `w:before` 应至少是 `w:after` 的 2 倍。常见好值：before=360 / after=120，或 before=240 / after=80。
3. **列表检查**：列表项的 `w:after` 应小于正文 `w:after` 的一半。若正文用 160，列表项应用 40-60。

## 4. 对齐与网格

### 为何有效

对齐创造眼睛沿页面向下跟随的隐形线。当元素共享同一边缘，读者感知到秩序和意图。当元素略微错位（差几 twips），页面看起来潦草，即便读者无法有意识地识别原因。

**左对齐 vs 两端对齐：**

- **左对齐**（右参差）最适合英文和其他拉丁文字语言。不齐的右边实际上有助于阅读，因为每行有独特轮廓，使眼睛更易找到下一行。两端对齐强制不均匀的词间距，产生穿过段落纵向流动的 distracting "白河"。

- **两端对齐**最适合 CJK 文本。中文、日文、韩文字符设计上等宽 — 每个占据隐形网格中相同单元。两端对齐完美保留此网格。CJK 文本右参差会破坏网格，看起来不整洁。

**缩进规则**：用首行缩进或段间距分隔段落 — 绝不同时用。它们服务于同一目的（标记段落边界）。同时用既浪费空间又造成视觉结巴。

- 西方惯例：段间距（无缩进）更现代。
- CJK 惯例：2 字符首行缩进是标准。
- 学术惯例：0.5 英寸首行缩进是传统。

### 好的示例

```xml
<!-- 英文正文：左对齐，段间距，无缩进 -->
<w:pPr>
  <w:jc w:val="left"/>
  <w:spacing w:after="160" w:line="276" w:lineRule="auto"/>
  <!-- 无 w:ind firstLine -->
</w:pPr>

<!-- CJK 正文：两端对齐，首行缩进 2 字符，无段间距 -->
<w:pPr>
  <w:jc w:val="both"/>
  <w:spacing w:after="0" w:line="360" w:lineRule="auto"/>
  <w:ind w:firstLineChars="200"/>
</w:pPr>

<!-- 制表位创建对齐的列 -->
<w:pPr>
  <w:tabs>
    <w:tab w:val="left" w:pos="2880"/>   <!-- 2 英寸 -->
    <w:tab w:val="right" w:pos="9360"/>  <!-- 6.5 英寸（右边距） -->
  </w:tabs>
</w:pPr>
```

```
  英文段落分隔（好 — 间距，无缩进）：

  这是第一段，带一些文本
  自然换行到第二行。

  这是第二段。上方的间隙
  清晰标记了边界。


  CJK 段落分隔（好 — 缩进，无间距）：

  　　第一段正文内容从这里开始，使用两个字符
  的首行缩进来标记段落边界。
  　　第二段紧跟其后，没有段间距，但首行缩进
  清晰地标识了新段落的开始。
```

### 坏的示例

```xml
<!-- 英文正文：两端对齐（产生词距白河） -->
<w:pPr>
  <w:jc w:val="both"/>
  <w:spacing w:after="160" w:line="276" w:lineRule="auto"/>
  <w:ind w:firstLine="720"/>  <!-- 既缩进又间距：冗余 -->
</w:pPr>

<!-- CJK 正文：左对齐（破坏字符网格） -->
<w:pPr>
  <w:jc w:val="left"/>
  <w:spacing w:after="200" w:line="276" w:lineRule="auto"/>
  <!-- 无缩进，改用间距 — 对 CJK 不地道 -->
</w:pPr>
```

问题：
- 两端对齐的英文文本配窄栏会产生不均匀词距。
- 同时用首行缩进和段间距是冗余的。
- 左对齐的 CJK 破坏了 CJK 读者期望的字符网格。
- 用间距分隔的 CJK 看起来像翻译的西方版式。

### 快速测试

1. **白河测试**：在两端对齐的英文文本中，眯眼寻找穿过段落的纵向白色条纹。若看到，切换到左对齐或增加栏宽。
2. **双重信号检查**：文档是否同时用首行缩进和段间距？若是，移除一个。CJK/学术选缩进，现代西方选间距。
3. **制表位对齐**：若用制表位做列，文档中所有制表位是否用相同位置？不一致的制表位产生锯齿状的隐形网格线。

## 5. 重复与一致性

### 为何有效

一致性是信任信号。当读者看到每个 H2 都相同、每个表格遵循同一模式、每个页码都在同一位置，他们不自觉地信任文档是用心制作的。一处不一致 — 一个 H2 是 15pt 而非 14pt、一个表格有不同边框 — 就破坏了这种信任，让读者质疑内容。

一致性还降低认知负荷。一旦读者学到"粗体深蓝 = 章节标题"，他们就不再花精力识别结构，完全专注于内容。每处不一致都迫使他们重新评估："这是一种不同的标题，还是有人忘了应用样式？"

实现规则很简单：**用命名样式，而非直接格式。** 若你将 Heading2 定义为样式并处处应用，一致性是自动的。若你逐个手动设置字号、粗体、颜色，不一致不可避免。

### 好的示例

```xml
<!-- 在 styles.xml 中一次性定义样式 -->
<w:style w:type="paragraph" w:styleId="Heading2">
  <w:name w:val="heading 2"/>
  <w:basedOn w:val="Normal"/>
  <w:next w:val="Normal"/>
  <w:pPr>
    <w:keepNext/>
    <w:keepLines/>
    <w:spacing w:before="360" w:after="120"/>
    <w:outlineLvl w:val="1"/>
  </w:pPr>
  <w:rPr>
    <w:rFonts w:asciiTheme="majorHAnsi" w:hAnsiTheme="majorHAnsi"/>
    <w:b/>
    <w:sz w:val="32"/>
    <w:color w:val="1F3864"/>
  </w:rPr>
</w:style>

<!-- 一致应用：每个 H2 引用该样式 -->
<w:p>
  <w:pPr>
    <w:pStyle w:val="Heading2"/>
    <!-- 无直接格式覆盖 -->
  </w:pPr>
  <w:r><w:t>市场分析</w:t></w:r>
</w:p>
```

使用表格样式时，一次性定义并用于每个表格：

```xml
<!-- 所有表格引用同一样式 -->
<w:tblPr>
  <w:tblStyle w:val="GridTable4Accent1"/>
  <w:tblW w:w="0" w:type="auto"/>
</w:tblPr>
```

### 坏的示例

```xml
<!-- 第一个 H2：手动格式化 -->
<w:p>
  <w:pPr>
    <w:spacing w:before="360" w:after="120"/>
  </w:pPr>
  <w:r>
    <w:rPr>
      <w:b/>
      <w:sz w:val="32"/>
      <w:color w:val="1F3864"/>
    </w:rPr>
    <w:t>市场分析</w:t>
  </w:r>
</w:p>

<!-- 第二个 H2：略有不同（16pt 还是 16pt？不，15pt！） -->
<w:p>
  <w:pPr>
    <w:spacing w:before="240" w:after="160"/>  <!-- 间距不同！ -->
  </w:pPr>
  <w:r>
    <w:rPr>
      <w:b/>
      <w:sz w:val="30"/>   <!-- 15pt 而非 16pt！ -->
      <w:color w:val="2E74B5"/>  <!-- 蓝色色调不同！ -->
    </w:rPr>
    <w:t>财务概览</w:t>
  </w:r>
</w:p>
```

问题：
- 无样式引用 — 一切都是直接格式。
- 第二个 H2 字号（30 vs 32）、颜色、间距都不同。
- 若有 20 个标题，每个都可能略有漂移。
- 日后改设计意味着逐个编辑每个标题。

### 快速测试

1. **样式审计**：每个段落是否引用 `w:pStyle`？若发现只有直接格式无样式的段落，那是一致性风险。
2. **搜索差异**：在 XML 中搜索与 `w:b`（粗体）一起使用的所有 `w:sz` 值。若本应同级的标题找到三个不同字号，存在不一致。
3. **表格检查**：文档中所有表格是否引用同一 `w:tblStyle`？若某些表格手动定义边框而其他用样式，文档会显得斑驳。
4. **页码**：检查页眉/页脚内容是否定义在默认节属性中并被所有节继承，而非每节不一致地重新定义。

## 6. 视觉层级与流向

### 为何有效

设计良好的文档引导读者眼睛沿可预测路径：顶部标题、下方副标题、作为路标的章节标题、作为主要内容的正文、作为支撑细节的脚注和题注。此流向映射阅读优先级 — 最重要的信息视觉上最突出。

层级中每级必须**与相邻级可区分**。H1 不同于正文还不够；H1 还必须明显不同于 H2，H2 不同于 H3。若任何两个相邻级太相似，层级在该点崩塌。

有效层级使用**多个同时信号**：

| 级别    | 字号  | 字重  | 颜色   | 上方间距 |
|----------|-------|---------|---------|---------------|
| 标题    | 26pt  | 粗体    | #1F3864 | 0（顶部）       |
| 副标题 | 15pt  | 常规 | #4472C4 | 4pt           |
| H1       | 20pt  | 粗体    | #1F3864 | 24pt          |
| H2       | 16pt  | 粗体    | #1F3864 | 18pt          |
| H3       | 13pt  | 粗体    | #1F3864 | 12pt          |
| 正文     | 11pt  | 常规 | #333333 | 0pt           |
| 题注  | 9pt   | 斜体  | #666666 | 4pt           |
| 脚注 | 9pt   | 常规 | #666666 | 0pt           |

注意每级至少在两个维度上区别于相邻级（字号+字重，或字号+颜色，或字重+样式）。单维度差异脆弱，易被忽略。

**分节符**在长文档中创造节奏。每个主要章节（H1）前的分页给读者心理重置。章节内，一致的标题+正文模式创造可预测的节拍，使长文档不那么令人生畏。

### 好的示例

```xml
<!-- 标题：大、粗体、海军蓝、居中 -->
<w:style w:type="paragraph" w:styleId="Title">
  <w:pPr>
    <w:jc w:val="center"/>
    <w:spacing w:after="80"/>
  </w:pPr>
  <w:rPr>
    <w:b/>
    <w:sz w:val="52"/>
    <w:color w:val="1F3864"/>
  </w:rPr>
</w:style>

<!-- 副标题：中、常规字重、较浅蓝、居中 -->
<w:style w:type="paragraph" w:styleId="Subtitle">
  <w:pPr>
    <w:jc w:val="center"/>
    <w:spacing w:after="320"/>
  </w:pPr>
  <w:rPr>
    <w:sz w:val="30"/>
    <w:color w:val="4472C4"/>
  </w:rPr>
</w:style>

<!-- H1：段前分页，大粗体海军蓝 -->
<w:style w:type="paragraph" w:styleId="Heading1">
  <w:pPr>
    <w:pageBreakBefore/>
    <w:keepNext/>
    <w:keepLines/>
    <w:spacing w:before="480" w:after="160"/>
    <w:outlineLvl w:val="0"/>
  </w:pPr>
  <w:rPr>
    <w:b/>
    <w:sz w:val="40"/>
    <w:color w:val="1F3864"/>
  </w:rPr>
</w:style>

<!-- 题注：小、斜体、灰 -->
<w:style w:type="paragraph" w:styleId="Caption">
  <w:pPr>
    <w:spacing w:before="80" w:after="200"/>
  </w:pPr>
  <w:rPr>
    <w:i/>
    <w:sz w:val="18"/>
    <w:color w:val="666666"/>
  </w:rPr>
</w:style>
```

```
  视觉流向（好）：

  +----------------------------------+
  |                                  |
  |     2025 年度报告                |  <- 标题：26pt 粗体海军蓝居中
  |     Acme 公司                    |  <- 副标题：15pt 常规蓝
  |                                  |
  |                                  |
  +----------------------------------+

  +----------------------------------+
  |                                  |
  |  1. 执行摘要                     |  <- H1：20pt 粗体海军蓝（分页）
  |                                  |
  |  介绍年度主要发现的               |  <- 正文：11pt 常规灰
  |  正文。                          |
  |                                  |
  |  1.1 营收亮点                    |  <- H2：16pt 粗体海军蓝
  |                                  |
  |  营收同比增长 23%，               |  <- 正文
  |  受...驱动。                     |
  |                                  |
  |  图 1：营收增长                  |  <- 题注：9pt 斜体灰
  |                                  |
  +----------------------------------+

  每级立即可辨。眼睛自然流向
  标题 -> 标题 -> 正文 -> 题注。
```

### 坏的示例

```xml
<!-- 所有标题与正文同色，字号差异极小 -->
<w:style w:type="paragraph" w:styleId="Heading1">
  <w:rPr>
    <w:b/>
    <w:sz w:val="28"/>       <!-- 14pt — 仅比正文高 3pt -->
    <w:color w:val="000000"/> <!-- 与正文同色 -->
  </w:rPr>
</w:style>

<!-- 题注与正文同号，非斜体 -->
<w:style w:type="paragraph" w:styleId="Caption">
  <w:rPr>
    <w:sz w:val="22"/>        <!-- 与正文同 11pt！ -->
    <w:color w:val="000000"/> <!-- 与正文同色 -->
  </w:rPr>
</w:style>

<!-- 主要章节间无分页 -->
<!-- H1 无 pageBreakBefore、keepNext 或 keepLines -->
```

问题：
- H1 14pt 与正文 11pt 太近（比率 1.27 — 孤立看可接受，但配黑色匹配正文，层级很弱）。
- 题注与正文无法区分。
- 无分页意味着主要章节相互渗透，无视觉节奏。
- 一切都是黑色，颜色提供零层级信号。

### 快速测试

1. **眯眼测试**：看整页时模糊视线。你应看到 3-4 个不同"灰度级"。若页面看起来是统一色调，层级太扁。
2. **扫描测试**：快速翻页。每页能否在一秒内识别章节边界？若是，视觉层级有效。若页面模糊成片，H1 需更强区分。
3. **相邻级测试**：对每个标题级，检查它是否在以下至少 2 项上区别于下一级：字号、字重、颜色、样式（斜体）。单维度差异会丢失。
4. **节奏测试**：超过 10 页的文档中，主要章节（H1）是否从新页开始？若否，长文档会感觉是无差异的流。给 Heading1 加 `w:pageBreakBefore`。

## 总结：决策清单

拿不准排版选择时，过一遍这些检查：

| 原则 | 问题 | 若否... |
|-----------|----------|----------|
| 留白 | 页面是否至少有 30% 留白？ | 增加页边距或间距 |
| 对比 | 眯眼能否数出标题级数？ | 增大字号比率（目标 1.25x） |
| 邻近 | 每个标题是否明确属于下方文本？ | 使段前 > 段后（2:1） |
| 对齐 | 英文是否左对齐、CJK 是否两端对齐？ | 切换对齐模式 |
| 重复 | 所有同级元素是否用同一样式？ | 用样式替换直接格式 |
| 层级 | 一臂距离能否看到文档结构？ | 增加更多区分信号 |

**当两个原则冲突时，按此顺序优先：**

1. **可读性**（留白、行距）— 始终优先
2. **层级**（对比、比例）— 读者必须找到所需
3. **一致性**（重复）— 建立信任
4. **美学**（对齐、分组）— 收尾点缀
